Accessibility Note

This presentation uses an out-of-the box HTML slideshow script which by itself is inaccessible. Keybindings exist to traverse forward and backward with arrow keys, but focus never shifts as the slides advance. The slides themselves are well structured, so you can use a read all command to listen to everything linearly. I've also added a live region which should work well for screen readers that support live regions. As you traverse through the slides using arrow keys, the slide's content will be added to the live region. If all else fails, you can also access the slideshow content by turning off CSS and JavaScript.

Shadow DOM

Explaining the web

  • Goal: primitives which can "explain" how native elements work
  • Sufficient power to create existing elements, thus sufficient power to create new elements
  • Can then explain future web platform features in these terms
  • This is very much a work in progress, e.g. how do we explain <label>, or the focus style behavior of <button>?
  • ARIA very much has a role to play in this: this stretches ARIA far beyond its original use case

Work in progress...

  • Specs are still under discussion
    • What level of encapsulation should Shadow DOM provide?
    • How do we extend native elements?
  • Browser support for existing specs is... inconsistent
